## Root Files

### .noodlecrew.yml

**Purpose:** Crew configuration â€” experts, LLM settings, phases, gates.

**Created by:** `ncrew init`

**Modified by:** User (manual editing)

See [Configuration Schema](configuration-schema.md) for complete specification.

```yaml
project:
  name: "my-project"
  type: "saas"

crew:
  experts:
    - product-owner
    - software-architect
    - developer
  llm:
    provider: "claude"
    model: "claude-sonnet-4-5"
    max_iterations: 100
    max_cost: 30.00

phases:
  - discovery
  - architecture
  - implementation

validation:
  human_gates:
    - architecture
```

## Directory Numbering Convention

Directories are numbered to indicate execution order:

| Number | Phase | Expert |
| ------ | ----- | ------ |
| 00 | Input | User |
| 01 | Discovery | Product Owner |
| 02 | Planning | Product Manager (future) |
| 03 | Design | UX Designer (future) |
| 04 | Architecture | Software Architect |
| 05 | Implementation | Developer |
| 06 | QA | QA Engineer (future) |
| 07 | Delivery | â€” (future) |

Gaps (02, 03, 06, 07) are reserved for future phases.

## Git Integration

Every iteration creates a git commit:

```
feat(discovery): generate PRD (iteration 1)
feat(discovery): define user personas (iteration 2)
feat(architecture): ADR-001 frontend stack (iteration 3)
...
```

**Commit format:** `feat(<phase>): <task description> (iteration <n>)`

**Tags:** Created at phase completion (e.g., `v0.1.0-discovery-complete`)
